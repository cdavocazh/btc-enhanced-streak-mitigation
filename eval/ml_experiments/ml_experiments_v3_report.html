4. The Aggression Problem — Hard Gate vs Soft Signal

Hard Gate (B) — What Went Wrong
Trades taken38 / 54
Trades rejected16 (30%)
Rejected wins (missed)6
Rejected losses (avoided)10
Rejection accuracy62.5%

The hard gate blocks 30% of trades. While 62.5% of rejections are correct (losses), the missed winners include large-PnL trades (Sep $40k, Jan $41k months) that dwarf the avoided losses. With an asymmetric payout (TP=4.5x ATR vs SL=1.8x ATR), each missed winner costs ~2.5x more than each avoided loss saves.

Soft Signal (E, F) — The Fix
Trades taken54 / 54 (100%)
Trades rejected0
Avg position scale0.62–0.65
Low-confidence reduction0.3x–0.5x
High-confidence boost1.0x–1.2x
PnL retained vs baseline82–84%
DD reduction vs baseline-33% to -35%

Soft-signal strategies never reject trades. Instead, they scale position size based on ML confidence. Low-confidence setups get 0.3x–0.5x the normal position; high-confidence get 1.0x–1.2x. This preserves exposure to winning trades while reducing risk on likely losers.

Why soft beats hard with asymmetric payouts: The strategy's TP/SL ratio is 4.5:1.8 = 2.5:1. Each winning trade generates ~2.5x the dollar amount of each losing trade. When a hard gate blocks a winner, it costs $2.50 in expected value per $1.00 of avoided loss. At 62.5% rejection accuracy (10L blocked vs 6W blocked), the net is: 10 × $1 saved − 6 × $2.5 missed = −$5 net. Hard gates are only profitable if rejection accuracy exceeds ~71%.

Key Conclusions

  1. Hard rejection gates are counterproductive for strategies with asymmetric payouts (TP > SL). Missed winners cost more than avoided losses save.
  2. Soft-signal scaling is the correct ML approach. Map ML confidence to position size (0.3x–1.2x) instead of binary accept/reject.
  3. Single-layer ML outperforms multi-layer stacking. Each additional soft layer compounds scale compression without adding discriminative value. One model at 0.62x avg scale beats four models at 0.35x.
  4. XGBoost and LightGBM perform nearly identically as soft scalers (Calmar 2.23 vs 2.22). Either can be used; XGB has a slight edge in profit factor.
  5. No ML model achieves meaningful sizing differentiation between winners and losers. The benefit comes from overall exposure reduction, not selective sizing. This suggests the trade-level features available (positioning, RSI, ADX, etc.) do not reliably predict individual trade outcomes — only broad market conditions.
  6. The RF regime classifier over-fits to "RangingLowVol" as unfavorable, but this regime represents ~46% of entries and still has positive expected value given the payout ratio.
  7. More training data helps modestly. Expanding from 109 entry-only samples to 114k+ all-bar samples improved the hard gate's WR from 31.2% to 28.9% (worse) but improved soft strategies' Calmar ratios by providing better calibration.

Implementation Recommendations

Do
  • Use XGBoost win probability as a soft position scaler
  • Scale range: 0.3x (prob=0.15) to 1.2x (prob=0.55+)
  • Retrain monthly with walk-forward on all-bar data
  • Keep all existing strategy entry filters unchanged
  • Maintain tiered risk + streak mitigation on top of ML scaling
Don't
  • Don't use hard rejection gates (binary accept/reject)
  • Don't stack multiple ML layers multiplicatively
  • Don't use RF regime as a trade filter (only for context)
  • Don't expect ML to predict individual trade outcomes
  • Don't trust ML sizing differentiation with <100 eval trades
