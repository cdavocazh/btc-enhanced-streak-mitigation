LaunchD Setup Guide - BTC Data Refresh

This guide explains how to set up automatic data refresh for Binance Futures data using macOS LaunchD. The system will automatically fetch new data every 5 minutes and fix any data corruption errors.

Overview

Features

  • Automatic Updates: Runs every 5 minutes via LaunchD
  • Auto-Recovery: Validates and fixes corrupted CSV files before each run
  • Append-Only: Never deletes existing data, only appends new data
  • Rate Limit Aware: Respects Binance API rate limits (900 requests/5min)
  • Detailed Logging: Separate logs for stdout and stderr

File Structure

binance-futures-data/ ├── extract_binance_data.py # Main extraction logic ├── refresh_data.py # Refresh script (quick/full modes) ├── launchd_refresh.sh # Shell wrapper for LaunchD ├── com.btc.datarefresh.plist # LaunchD configuration ├── logs/ │ ├── launchd.log # Standard output log │ └── launchd_error.log # Error log └── data/ ├── price.csv ├── top_trader_position_ratio.csv ├── top_trader_account_ratio.csv ├── global_ls_ratio.csv ├── open_interest.csv ├── funding_rate.csv └── last_timestamps.json

Installation Steps

1

Update Python Path

Edit launchd_refresh.sh to update the Python path if needed:

# Find your Python path
which python3

# Edit the script
nano launchd_refresh.sh

# Update this line with your Python path:
PYTHON_PATH="/Users/kris.zhang/mambaforge/bin/python3"
2

Make Script Executable

chmod +x launchd_refresh.sh
3

Create Logs Directory

mkdir -p logs
4

Update Plist Paths

Edit com.btc.datarefresh.plist to update paths if your username or directory is different:

# Edit the plist file
nano com.btc.datarefresh.plist

# Update these paths:
# - ProgramArguments (path to launchd_refresh.sh)
# - WorkingDirectory (path to binance-futures-data)
# - StandardOutPath (path to launchd.log)
# - StandardErrorPath (path to launchd_error.log)
# - PATH in EnvironmentVariables
5

Copy Plist to LaunchAgents

cp com.btc.datarefresh.plist ~/Library/LaunchAgents/
6

Load the LaunchD Agent

launchctl load ~/Library/LaunchAgents/com.btc.datarefresh.plist
Success! The data refresh will now run automatically every 5 minutes.

Auto-Recovery Features

CSV Validation & Auto-Fix

The system automatically detects and fixes common data issues:

  • Empty Files: Detected and skipped to prevent crashes
  • Malformed Timestamps: Patterns like "1-03 23:45:00+00:00" are fixed to "2026-01-03 23:45:00+00:00"
  • Duplicate Rows: Automatically removed, keeping the latest version
  • Unparseable Rows: Invalid rows are removed to maintain data integrity
  • Out-of-Order Data: Data is sorted by timestamp after each operation

Error Recovery Flow

  1. LaunchD triggers launchd_refresh.sh every 5 minutes
  2. Shell script calls refresh_data.py
  3. validate_all_csv_files() is called before any data extraction
  4. Any corrupted files are automatically fixed
  5. Data extraction proceeds normally
  6. Next run will work even if previous run crashed

Troubleshooting

LaunchD Not Running

# Check status
launchctl list | grep btc

# If not listed, reload:
launchctl unload ~/Library/LaunchAgents/com.btc.datarefresh.plist
launchctl load ~/Library/LaunchAgents/com.btc.datarefresh.plist

Python Not Found

# Find your Python path
which python3

# Update launchd_refresh.sh with the correct path
# Also update PATH in com.btc.datarefresh.plist

Permission Denied

# Make script executable
chmod +x launchd_refresh.sh

# Check file permissions
ls -la launchd_refresh.sh

No Log Output

# Check if logs directory exists
mkdir -p logs

# Check log file permissions
ls -la logs/

# Run manually to test
./launchd_refresh.sh
Note

LaunchD agents only run when the user is logged in. For running 24/7 (even when logged out), use a LaunchDaemon instead (requires root privileges and different plist location).
