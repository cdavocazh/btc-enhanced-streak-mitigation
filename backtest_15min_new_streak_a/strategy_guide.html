Adaptive Streak Reduction - Strategy Guide

Generated: 2026-01-29 | Version: Adaptive_TieredStreak_AHours

Overview

This guide documents the Adaptive Streak Reduction strategies designed to minimize losing streaks while maintaining profitability. These strategies build upon the original Tiered Streak Mitigation framework by adding two key adaptive filters:

  1. ADX Filter (Market Regime Detection) - Avoids trades in ranging/choppy markets
  2. Progressive Positioning Threshold - Increases entry requirements after consecutive losses

1. Indicator Calculations

1.1 ADX (Average Directional Index)

Purpose: Measures trend strength regardless of direction. Used to filter out ranging markets.

Parameters: Period = 14

Calculation Steps: 1. Calculate True Range (TR): TR = max(high - low, abs(high - prev_close), abs(low - prev_close)) 2. Calculate Directional Movement: +DM = high - prev_high (if positive and > -(low - prev_low), else 0) -DM = prev_low - low (if positive and > +(high - prev_high), else 0) 3. Smooth using Wilder's smoothing (EMA with alpha = 1/14): ATR_14 = EMA(TR, 14) +DI_14 = 100 * EMA(+DM, 14) / ATR_14 -DI_14 = 100 * EMA(-DM, 14) / ATR_14 4. Calculate DX and ADX: DX = 100 * abs(+DI_14 - -DI_14) / (+DI_14 + -DI_14) ADX = EMA(DX, 14)

Interpretation:

  • ADX < 20: Weak/No trend (ranging market) - AVOID TRADING
  • ADX 20-25: Trend emerging
  • ADX 25-50: Strong trend - OPTIMAL FOR TRADING
  • ADX > 50: Very strong trend (potentially overbought)

1.2 ATR (Average True Range)

Purpose: Measures volatility for position sizing and stop-loss placement.

TR = max(high - low, abs(high - prev_close), abs(low - prev_close)) ATR_14 = SMA(TR, 14) ATR_5 = SMA(TR, 5) # Short-term ATR for adaptive SL # Adaptive ATR (used in Sequential_Regime): Adaptive_ATR = min(ATR_5, ATR_14) # More conservative

1.3 Positioning Indicators

Source: Binance Futures API - Top Trader Position Ratio

# Raw positioning data (0-1 range, 0.5 = neutral): pos_long = Top Trader Long Position / (Long + Short) # Normalized positioning (-1 to +1): pos_normalized = (pos_long - 0.5) * 2 # Positioning momentum (rate of change): pos_roc = pos_long.pct_change(periods=4) # 1-hour lookback

Interpretation:

  • pos_long > 0.5: Majority of top traders are long (bullish sentiment)
  • pos_long < 0.5: Majority of top traders are short (bearish sentiment)
  • pos_roc > 0: Positioning momentum increasing (strengthening conviction)

1.4 Volume Indicators

# Volume ratio (current vs average): vol_ma = SMA(volume, 20) vol_ratio = volume / vol_ma # Volume surge detection: vol_std = std(volume, 20) vol_surge = volume > (vol_ma + 2 * vol_std)

Adaptive Volume Threshold (based on ADX):

  • Trending market (ADX ≥ 20): vol_ratio ≥ 0.7 required
  • Ranging market (ADX < 20): vol_ratio ≥ 1.2 required (higher bar)

1.5 RSI (Relative Strength Index)

# Calculate gains and losses: delta = close - prev_close gain = max(delta, 0) loss = max(-delta, 0) # Smooth gains and losses: avg_gain = EMA(gain, 14) avg_loss = EMA(loss, 14) # Calculate RSI: RS = avg_gain / avg_loss RSI = 100 - (100 / (1 + RS))

Usage: RSI < 30 for oversold (long entry), RSI > 70 for overbought (exit consideration)

2.2 Adaptive Entry Filters

Filter 1: ADX Market Regime Filter

# Only enter in trending markets if adx_filter_enabled: if ADX < 20: BLOCK_ENTRY # Ranging market - too risky else: ALLOW_ENTRY # Trending market - proceed

Rationale: Historical analysis showed 45% of losing streaks occurred during choppy, range-bound markets where ADX was below 20.

Filter 2: Progressive Positioning Threshold

# Threshold increases after each consecutive loss PROGRESSIVE_THRESHOLDS = { 0: 0.4, # No recent losses - standard threshold 1: 0.5, # 1 consecutive loss - slightly higher 2: 0.6, # 2 consecutive losses - higher 3: 0.8, # 3 consecutive losses - much higher 4: 1.0, # 4+ losses - effectively blocked } consecutive_losses = count_recent_consecutive_losses() required_pos_long = PROGRESSIVE_THRESHOLDS.get(consecutive_losses, 1.0) if pos_long < required_pos_long: BLOCK_ENTRY # Positioning not strong enough else: ALLOW_ENTRY

Rationale: After losses, require stronger conviction signals before re-entering. This prevents "revenge trading" and allows market conditions to improve.

Filter 3: Cooldown Period

COOLDOWN_TRIGGER = 5 # Trigger after 5 consecutive losses COOLDOWN_BARS = 96 # 24 hours in 15-min bars if consecutive_losses >= COOLDOWN_TRIGGER: if bars_since_last_loss < COOLDOWN_BARS: BLOCK_ENTRY # Still in cooldown else: ALLOW_ENTRY # Cooldown expired

Rationale: Extended losing streaks indicate significant regime change. Force a pause to let market conditions stabilize.

Filter 4: Higher Highs/Higher Lows (HH/HL)

# Used in Sequential_Regime strategy lookback = 10 # bars recent_highs = high.rolling(lookback).max() recent_lows = low.rolling(lookback).min() # Check for uptrend structure higher_high = high > recent_highs.shift(1) higher_low = low > recent_lows.shift(1) if hh_hl_filter_enabled: if not (higher_high or higher_low): BLOCK_ENTRY # No clear uptrend structure

Filter 5: Positioning Momentum

# Rate of change in positioning over 4 bars (1 hour) pos_roc = (pos_long - pos_long.shift(4)) / pos_long.shift(4) if pos_momentum_filter_enabled: if pos_roc < 0: BLOCK_ENTRY # Positioning momentum declining

3. Exit Conditions

3.1 Stop Loss

# Standard Stop Loss sl_multiplier = 2.0 # Configurable stop_loss = entry_price - (ATR_14 * sl_multiplier) # Adaptive Stop Loss (Sequential_Regime) adaptive_atr = min(ATR_5, ATR_14) stop_loss = entry_price - (adaptive_atr * sl_multiplier)

Rationale: Adaptive ATR uses the smaller of short-term and long-term ATR, providing tighter stops when volatility is compressed.

3.2 Take Profit

# Standard Take Profit tp_multiplier = 3.0 # 1.5:1 risk-reward ratio take_profit = entry_price + (ATR_14 * tp_multiplier) # Multi-TP (for Conservative strategies) if multi_tp_enabled: tp1_pct = 0.30 # Take 30% at first target tp1 = entry_price + (ATR_14 * tp_multiplier * 0.5) # Half distance tp2 = entry_price + (ATR_14 * tp_multiplier) # Full distance
