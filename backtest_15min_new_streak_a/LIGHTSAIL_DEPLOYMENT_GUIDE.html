3. Lightsail Instance Setup

1

Create Lightsail Instance

Go to AWS Lightsail Console

  • Click "Create instance"
  • Select region closest to you (or your users)
  • Platform: Linux/Unix
  • Blueprint: OS Only → Ubuntu 24.04 LTS (recommended) or 22.04 LTS
  • Instance plan: $10/month (2GB RAM, 2 vCPU)
  • Name your instance: btc-dashboard
2

Configure Networking

After instance is created, go to Networking tab:

  • Create Static IP (free while attached)
  • Attach static IP to your instance

Add firewall rules:

Application Protocol Port
SSH TCP 22
HTTP TCP 80
HTTPS TCP 443
Custom (Streamlit) TCP 8501
3

Connect via SSH

Option 1: Use Lightsail browser-based SSH (click "Connect using SSH" button)

Option 2: Download SSH key and connect from terminal:

# Download your SSH key from Lightsail console
# Then connect:
chmod 400 ~/Downloads/LightsailDefaultKey-*.pem
ssh -i ~/Downloads/LightsailDefaultKey-*.pem ubuntu@YOUR_STATIC_IP

4. Server Configuration

4.1 Update System

# Update package lists and upgrade
sudo apt update && sudo apt upgrade -y

# Install essential packages
sudo apt install -y python3-pip python3-venv git nginx certbot python3-certbot-nginx

# Install build dependencies for Python packages
sudo apt install -y build-essential python3-dev

4.2 Create Application User (Optional but Recommended)

# You can use the default ubuntu user, or create a dedicated user:
# sudo adduser btcapp
# sudo usermod -aG sudo btcapp

# For this guide, we'll use ubuntu user
cd /home/ubuntu

4.3 Set Up Python Virtual Environment

# Create virtual environment
python3 -m venv ~/btc-venv

# Activate it
source ~/btc-venv/bin/activate

# Upgrade pip
pip install --upgrade pip

# Add to bashrc for auto-activation
echo 'source ~/btc-venv/bin/activate' >> ~/.bashrc

4.4 Configure Swap (Recommended for 1-2GB instances)

# Create 2GB swap file
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# Make permanent
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# Verify
free -h

6. Data Pipeline Setup (Cron)

6.1 Create Refresh Script

# Create the refresh script
cat > ~/btc-enhanced-streak-mitigation/binance-futures-data/cron_refresh.sh << 'EOF'
#!/bin/bash
# BTC Data Refresh Script for Cron

LOG_DIR="/home/ubuntu/btc-enhanced-streak-mitigation/binance-futures-data/logs"
SCRIPT_DIR="/home/ubuntu/btc-enhanced-streak-mitigation/binance-futures-data"
VENV="/home/ubuntu/btc-venv/bin/python3"

# Ensure log directory exists
mkdir -p "$LOG_DIR"

# Log start
echo "=========================================="  >> "$LOG_DIR/cron.log"
echo "Data Refresh Started: $(date)"             >> "$LOG_DIR/cron.log"
echo "=========================================="  >> "$LOG_DIR/cron.log"

# Run refresh
cd "$SCRIPT_DIR"
$VENV refresh_data.py >> "$LOG_DIR/cron.log" 2>&1

# Log end
echo "Data Refresh Ended: $(date)"               >> "$LOG_DIR/cron.log"
echo ""                                           >> "$LOG_DIR/cron.log"
EOF

# Make executable
chmod +x ~/btc-enhanced-streak-mitigation/binance-futures-data/cron_refresh.sh

6.2 Set Up Cron Job

# Edit crontab
crontab -e

# Add these lines (runs every 5 minutes):
*/5 * * * * /home/ubuntu/btc-enhanced-streak-mitigation/binance-futures-data/cron_refresh.sh

# Optional: Add weekly evaluation (Sundays at 2am)
0 2 * * 0 /home/ubuntu/btc-enhanced-streak-mitigation/eval/eval_regular_exec.sh >> /home/ubuntu/btc-enhanced-streak-mitigation/eval/logs/weekly.log 2>&1

6.3 Verify Cron is Running

# Check cron service
sudo systemctl status cron

# View cron logs
grep CRON /var/log/syslog | tail -20

# Check our refresh logs
tail -50 ~/btc-enhanced-streak-mitigation/binance-futures-data/logs/cron.log
The data pipeline will now automatically refresh every 5 minutes, keeping your dashboard data up-to-date.

7. Streamlit Dashboard Deployment

7.1 Create Streamlit Configuration

# Create Streamlit config directory
mkdir -p ~/.streamlit

# Create config file
cat > ~/.streamlit/config.toml << 'EOF'
[server]
headless = true
address = "0.0.0.0"
port = 8501
enableCORS = false
enableXsrfProtection = true

[browser]
gatherUsageStats = false

[theme]
primaryColor = "#ff9900"
backgroundColor = "#1a1a2e"
secondaryBackgroundColor = "#16213e"
textColor = "#e0e0e0"
EOF

7.2 Create Systemd Service

# Create service file
sudo cat > /etc/systemd/system/streamlit-dashboard.service << 'EOF'
[Unit]
Description=BTC Adaptive Strategy Dashboard
After=network.target

[Service]
Type=simple
User=ubuntu
Group=ubuntu
WorkingDirectory=/home/ubuntu/btc-enhanced-streak-mitigation
Environment="PATH=/home/ubuntu/btc-venv/bin"
ExecStart=/home/ubuntu/btc-venv/bin/streamlit run backtest_15min_new_streak_a/dashboard_adaptive.py --server.port 8501
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

# Reload systemd and enable service
sudo systemctl daemon-reload
sudo systemctl enable streamlit-dashboard
sudo systemctl start streamlit-dashboard

# Check status
sudo systemctl status streamlit-dashboard

7.3 Test Dashboard Access

8. Nginx & SSL Configuration

8.1 Configure Nginx Reverse Proxy

# Create Nginx config
sudo cat > /etc/nginx/sites-available/btc-dashboard << 'EOF'
server {
    listen 80;
    server_name YOUR_DOMAIN.com;  # Or use _ for IP-only access

    location / {
        proxy_pass http://localhost:8501;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400;
    }

    # WebSocket support for Streamlit
    location /_stcore/stream {
        proxy_pass http://localhost:8501/_stcore/stream;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400;
    }
}
EOF

# Enable the site
sudo ln -s /etc/nginx/sites-available/btc-dashboard /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default

# Test config
sudo nginx -t

# Reload Nginx
sudo systemctl reload nginx

8.2 Set Up SSL with Let's Encrypt (If Using Domain)

# Replace YOUR_DOMAIN.com with your actual domain
# Make sure DNS is pointing to your Lightsail IP first!

sudo certbot --nginx -d YOUR_DOMAIN.com

# Follow the prompts:
# - Enter email for renewal notices
# - Agree to terms
# - Choose to redirect HTTP to HTTPS (recommended)

# Verify auto-renewal
sudo certbot renew --dry-run

8.3 For IP-Only Access (No Domain)

# If not using a domain, access via:
# http://YOUR_STATIC_IP

# For self-signed SSL (not recommended for production):
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
    -keyout /etc/ssl/private/nginx-selfsigned.key \
    -out /etc/ssl/certs/nginx-selfsigned.crt

# Then update Nginx config to use these certificates
SSL is strongly recommended for any production deployment. Let's Encrypt provides free SSL certificates.

9.4 Backup Strategy

# Create backup script
cat > ~/backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/home/ubuntu/backups"
DATE=$(date +%Y%m%d)
mkdir -p $BACKUP_DIR

# Backup data files
tar -czf $BACKUP_DIR/data_$DATE.tar.gz \
    ~/btc-enhanced-streak-mitigation/binance-futures-data/data/

# Backup results
tar -czf $BACKUP_DIR/results_$DATE.tar.gz \
    ~/btc-enhanced-streak-mitigation/backtest_15min_new_streak_a/results/ \
    ~/btc-enhanced-streak-mitigation/eval/

# Keep only last 7 days
find $BACKUP_DIR -name "*.tar.gz" -mtime +7 -delete

echo "Backup complete: $DATE"
EOF

chmod +x ~/backup.sh

# Add to cron for daily backups
# 0 3 * * * /home/ubuntu/backup.sh

10. Troubleshooting

Common Issues & Solutions

❌ Dashboard Not Loading

# Check if Streamlit is running
sudo systemctl status streamlit-dashboard

# Check logs
sudo journalctl -u streamlit-dashboard -n 50

# Restart service
sudo systemctl restart streamlit-dashboard

# Check if port is in use
sudo lsof -i :8501

❌ Data Not Updating

# Check cron logs
tail -100 ~/btc-enhanced-streak-mitigation/binance-futures-data/logs/cron.log

# Run refresh manually
cd ~/btc-enhanced-streak-mitigation/binance-futures-data
~/btc-venv/bin/python refresh_data.py

# Check file timestamps
ls -la ~/btc-enhanced-streak-mitigation/binance-futures-data/data/

# Check cron is running
sudo systemctl status cron
crontab -l

❌ 502 Bad Gateway (Nginx)

# Check Nginx error log
sudo tail -50 /var/log/nginx/error.log

# Verify Streamlit is running on port 8501
curl -I http://localhost:8501

# Check Nginx config
sudo nginx -t

# Restart both services
sudo systemctl restart streamlit-dashboard
sudo systemctl restart nginx

❌ Out of Memory

# Check memory usage
free -h

# Check what's using memory
ps aux --sort=-%mem | head -10

# Add more swap if needed
sudo fallocate -l 4G /swapfile2
sudo chmod 600 /swapfile2
sudo mkswap /swapfile2
sudo swapon /swapfile2

❌ SSL Certificate Issues

# Check certificate status
sudo certbot certificates

# Renew certificate
sudo certbot renew

# Force renewal
sudo certbot renew --force-renewal
