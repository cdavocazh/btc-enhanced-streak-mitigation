Part 1: Indicator Calculations & Methodology

All indicators are computed from 15-minute OHLCV data and Binance positioning metrics. Below is the exact calculation methodology for each indicator used in the strategy.

1.1 Technical Indicators

Average True Range (ATR)

Purpose: Measures market volatility. Used for stop-loss and take-profit distance calculations.

Default Period: 14 (configurable: tested 14, 28, 42, 56)

# True Range = Maximum of: TR = max( high - low, # Current bar range abs(high - prev_close), # Gap up from previous close abs(low - prev_close) # Gap down from previous close ) # ATR = Simple Moving Average of True Range ATR = rolling_mean(TR, period=14)

Interpretation: Higher ATR = more volatile market = wider SL/TP distances. Lower ATR = tighter ranges.

Relative Strength Index (RSI)

Purpose: Identifies oversold conditions for pullback entries.

Default Period: 56 (configurable: tested 14, 28, 42, 56)

# Price change delta = close - prev_close # Separate gains and losses gain = rolling_mean(max(delta, 0), period=56) loss = rolling_mean(abs(min(delta, 0)), period=56) # RSI calculation RS = gain / loss RSI = 100 - (100 / (1 + RS))

Entry Range: 20 < RSI < 45 (pullback but not extreme oversold)

Simple Moving Average (SMA)

Purpose: Determines trend direction. Only long trades when price is above SMA.

Default Period: 200 (configurable: tested 50, 100, 150, 200)

SMA200 = rolling_mean(close, period=200) # Trend determination uptrend = close > SMA200 # Only take longs in uptrend

Bollinger Bands

Purpose: Identifies price deviation from mean. Used for context, not direct entry signals.

Period: 80 (fixed)

middle_band = rolling_mean(close, period=80) std_dev = rolling_std(close, period=80) upper_band = middle_band + (2 * std_dev) lower_band = middle_band - (2 * std_dev)

Pullback Percentage

Purpose: Measures how far price has pulled back from recent highs.

Lookback: 16 bars (4 hours)

recent_high = rolling_max(high, period=16) pullback_pct = (recent_high - close) / close * 100 # Entry filter: 0.5% to 3.0% pullback valid_pullback = (0.5 <= pullback_pct) and (pullback_pct <= 3.0)

Logic: We want a meaningful pullback (not chasing highs) but not too deep (potential trend reversal).

1.2 Volume Indicators

Volume Ratio

Purpose: Compares current volume to recent average. Identifies high-conviction moves.

vol_ma = rolling_mean(volume, period=96) # 24 hours of 15-min bars vol_ratio = volume / vol_ma # Interpretation: # vol_ratio > 1.5 = High volume (strong conviction) # vol_ratio > 1.0 = Above average # vol_ratio < 0.8 = Low volume (weak conviction)

Volume Trend

Purpose: Determines if volume is increasing or decreasing over time.

vol_ma_4h = rolling_mean(volume, period=16) # 4-hour average vol_trend = vol_ma_4h - shift(vol_ma_4h, 4) # Change vs 1 hour ago volume_increasing = vol_trend > 0

Volume Quality Score (0-2 scale)

Purpose: Composite score combining volume metrics for position sizing adjustment.

vol_score = 0 # Base volume ratio scoring if vol_ratio > 1.5: vol_score += 1.0 elif vol_ratio > 1.2: vol_score += 0.7 elif vol_ratio > 1.0: vol_score += 0.5 elif vol_ratio > 0.8: vol_score += 0.3 # Volume trend bonus if volume_increasing: vol_score += 0.5 # Bullish volume bonus (price up + good volume) if close > open and vol_ratio > 1.0: vol_score += 0.5 vol_score = min(vol_score, 2.0) # Cap at 2.0

1.3 Positioning Indicators (Binance Data)

Positioning Score

Purpose: Composite score based on Binance top trader positioning, funding rate, and global long/short ratio. Identifies smart money positioning.

pos_score = 0 # Top Trader Position Ratio (most important) if top_trader_long_pct > 60: pos_score += 1.5 # Strong bullish elif top_trader_long_pct > 55: pos_score += 1.0 # Bullish elif top_trader_short_pct > 60: pos_score -= 1.5 # Strong bearish elif top_trader_short_pct > 55: pos_score -= 1.0 # Bearish # Top Trader Account Ratio (secondary) if account_long_pct > 55: pos_score += 0.25 elif account_short_pct > 55: pos_score -= 0.25 # Global Long/Short Ratio (contrarian) if global_ls_ratio < 0.7: pos_score += 0.5 # Crowd is short = bullish elif global_ls_ratio > 1.5: pos_score -= 0.5 # Crowd is long = bearish # Funding Rate (contrarian) if funding_rate > 0.0005: pos_score -= 0.5 # Expensive to be long elif funding_rate < -0.0005: pos_score += 0.5 # Cheap to be long

Entry Threshold: Positioning score >= 0.4 (increases to 0.75 after 3+ consecutive losses)

Part 2: Entry Signal Logic

The strategy is long-only, entering on pullbacks during uptrends when top trader positioning is favorable.

2.1 Entry Conditions (All Must Be True)

  1. Uptrend Check: close > SMA200 - Price must be above the 200-period SMA
  2. Positioning Score: positioning_score >= 0.4 - Top traders must be net long
  3. RSI Filter: 20 < RSI < 45 - Must be in pullback zone (not overbought)
  4. Pullback Range: 0.5% < pullback_pct < 3.0% - Meaningful but not excessive pullback
  5. Minimum Activity: abs(positioning_score) >= 0.15 - Some positioning signal present
  6. No Open Trade: Only one position at a time
  7. Entry Hours: 0-11 UTC (Asian hours) or 0-23 UTC (all hours)

2.2 Entry Flow Diagram

New 15-min Bar | v [Price > SMA200?] --No--> Skip (not uptrend) | Yes v [Positioning Score >= 0.4?] --No--> Skip (smart money not bullish) | Yes v [20 < RSI < 45?] --No--> Skip (not in pullback zone) | Yes v [0.5% < Pullback < 3%?] --No--> Skip (pullback too small or too large) | Yes v [Volume Filter Pass?] --No--> Skip (only for VolFilter/Conservative strategies) | Yes v [No Open Position?] --No--> Skip (already in trade) | Yes v ENTER LONG | +-- Calculate position size based on risk model +-- Set SL at entry - (ATR x 1.8) +-- Set TP at entry + (ATR x 4.5 x adjustments)

Adaptive Entry Threshold

After 3+ consecutive losses, the positioning score threshold increases from 0.4 to 0.75. This prevents entering marginal setups during losing streaks, requiring stronger conviction before risking capital again.

Part 3: Exit Logic (Take Profit & Stop Loss)

3.1 Stop Loss (SL) Methodology

Base Stop Loss: ATR-Based

Multiplier: 1.8 x ATR below entry price

stop_price = entry_price - (ATR * 1.8) # Example: Entry at $50,000, ATR = $500 # Stop Loss = $50,000 - ($500 * 1.8) = $49,100

Dynamic SL Adjustments (Strategy-Specific)

1. Positioning Decline to Breakeven (PosVol_Combined, Conservative)

  • Trigger: Positioning score drops by > 0.4-0.5 points from entry
  • Action: Move SL to entry_price + (ATR x 0.1) (slight profit)
  • Rationale: Smart money exiting = reduce risk immediately

2. Volume Collapse SL Tightening (PosVol_Combined, VolFilter_Adaptive)

  • Trigger: vol_ratio < 0.5 AND bars_held > 4
  • Action: Tighten SL to close - (ATR x 1.0)
  • Rationale: Low volume = weak conviction = protect gains

3.2 Take Profit (TP) Methodology

Single TP Approach (Baseline, VolFilter_Adaptive)

Base Multiplier: 4.5 x ATR above entry price

# Base TP calculation tp_mult = 4.5 # Positioning score adjustments if positioning_score >= 1.5: tp_mult *= 1.30 # Very strong = 5.85x ATR elif positioning_score >= 1.0: tp_mult *= 1.15 # Strong = 5.175x ATR take_profit_price = entry_price + (ATR * tp_mult)

Multi-Level TP Approach (MultiTP_30, Conservative)

Purpose: Lock in partial profits early while letting remainder run.

# TP1: Partial exit (30-40% of position) tp1_price = entry_price + (ATR * 2.0) tp1_exit_pct = 0.30 # or 0.40 for Conservative # TP2: Remaining position (60-70%) tp2_price = entry_price + (ATR * tp_mult) # 4.5x adjusted # Execution flow: # 1. Price hits TP1 -> Exit 30% at tp1_price # 2. Remaining 70% continues toward TP2 # 3. If SL hit before TP2 -> Exit remaining at SL

3.3 Positioning Exit

Emergency Exit on Positioning Reversal

If positioning score drops below 0.0 (smart money turns bearish), exit entire position immediately at market price, regardless of P&L. This overrides both TP and SL.

if positioning_score < 0.0: exit_position(reason="pos_exit")

3.4 Exit Flow Summary

Every 15-min Bar (while in position): | v [Price <= Stop Loss?] --Yes--> EXIT (loss) | No v [Positioning Score < 0?] --Yes--> EXIT (positioning reversal) | No v [Dynamic SL Triggered?] --Yes--> Adjust SL (tighten) | No v [Price >= TP1?] --Yes--> Partial EXIT (30-40%, MultiTP only) | No v [Price >= Main TP?] --Yes--> EXIT (full profit) | No v HOLD POSITION

Part 4: Position Sizing Approaches

4.1 Fixed Risk Model (Asian Hours / All Hours)

Simple approach: risk a fixed dollar amount per trade.

BASE_RISK = $6,000 # Fixed risk per trade unit_risk = abs(entry_price - stop_price) position_size = BASE_RISK / unit_risk # Example: Entry $50,000, SL $49,100 (diff = $900) # Position = $6,000 / $900 = 6.67 BTC contracts

4.4 Additional Size Adjustments

Positioning Score Bonus:

  • If positioning_score >= 1.5: Multiply size by 1.2 (20% increase)
  • If positioning_score >= 1.0: Multiply size by 1.1 (10% increase)

Volume-Based Adjustment (VolFilter_Adaptive, Conservative):

  • If vol_ratio > 1.2: Multiply size by 1.25 (25% increase)
  • If vol_ratio < 0.8: Multiply size by 0.8 (20% decrease)
