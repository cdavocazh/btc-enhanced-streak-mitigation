Important: Understanding GitHub vs Runtime Traffic

A common question: "If GitHub restricts Binance API traffic, will cloud deployments work?"

┌─────────────────────────────────────────────────────────────────────────────┐ │ TRAFFIC FLOW CLARIFICATION │ ├─────────────────────────────────────────────────────────────────────────────┤ │ │ │ GitHub is ONLY used for: │ │ ┌──────────────────────────────────────────────────────────────────────┐ │ │ │ • Storing your source code │ │ │ │ • Triggering deployments (on git push) │ │ │ │ • NOT involved in any runtime API calls │ │ │ └──────────────────────────────────────────────────────────────────────┘ │ │ │ │ Runtime API calls go DIRECTLY from deployment server to Binance: │ │ │ │ ┌─────────────────┐ ┌─────────────────┐ │ │ │ Railway/Render │───── API Call ────▶│ Binance API │ │ │ │ AWS/GCP/VPS │ │ (fapi.binance) │ │ │ │ Local Laptop │◀──── Response ─────│ │ │ │ └─────────────────┘ └─────────────────┘ │ │ │ │ │ │ (serves dashboard) │ │ ▼ │ │ Your Browser │ │ │ └─────────────────────────────────────────────────────────────────────────────┘

Answer: Yes, all cloud solutions will work. The fetch script runs on the deployment server's infrastructure, making direct calls to Binance. GitHub has no involvement in runtime data fetching.

Note: For Railway/Render, choose EU server region if US IPs have restricted Binance access.

Option C: Python Background Daemon

#!/usr/bin/env python3 # fetch_daemon.py - Run with: python3 fetch_daemon.py import schedule import time import subprocess import logging logging.basicConfig(level=logging.INFO, format='%(asctime)s - %(message)s') def fetch_data(): logging.info("Fetching data from Binance...") subprocess.run(['python3', 'fetch_data.py'], cwd='/Users/kris.zhang/Github/btc-enhanced-streak-mitigation/binance-futures-data') # Run every 5 minutes schedule.every(5).minutes.do(fetch_data) fetch_data() # Run immediately while True: schedule.run_pending() time.sleep(1)

Important: Laptop Sleep/Power Considerations

All local solutions pause when your laptop sleeps or lid is closed. To maintain 24/7 data refresh:

  • macOS: Use caffeinate -i command or install "Amphetamine" app to prevent sleep
  • Windows: Adjust power settings to "Never sleep when plugged in"
  • Linux: Use systemd-inhibit or adjust power settings
  • Best solution: For 24/7 operation, use a Raspberry Pi ($50 one-time) or a VPS ($5/month)
